# 🧭 路线图

Unigen 的推进顺序：**先兑现确定性的用途，再逐步交给规划器。**

## 第一步：承载有实际意义的工作流（进行中）

优先落地一批真实价值的长链条工作流：

- 剧本 → 视频
- 创意/要求 → 小说
- 小说 → 剧本
- 需求 → 代码（可能聚焦细分类目，如 Web）


这一阶段以人工撰写为主、内建方法（Plan 等）辅助，同时验证平台本身：稳定 / 高效 / 灵活的运行，以及工作流发布能力。

## 第二步：规划器接手生成与改进

按 [[Prism（视角层）|prism]] → [[Loom（结构层过渡）|loom]] → [[HTN/HDDL（结构层完全体）|planner]] 推进，让工作流的产生、改进、维护尽可能由规划器完成。基于 OpenTelemetry 执行历史的持续改进也在这一阶段落地。

## 终极目标

**无人介入，全自动改进。** 更远处是认知模型 / 世界约束层（见 [[棱镜推理|prism]] 第九节）——让规划器知道"什么客观上做不到"，目前是开放研究。

## 最想要的社区反馈

- 用**复杂真实任务**验证或证伪 Prism 的"拉平"现象（[[棱镜推理|prism]] 第三、六节）；
- 可量化指标下的严格评测数据；
- Prism 明显无效或变差的任务类型——反例同样宝贵。

## 为什么反复纠缠 Prism 这个"小技巧"

因为 Prism 不是提质技巧，而是整个系统的最小逻辑单元。它做的事——把一个问题拆成若干正交视角、在每个视角内独立判断、再跨视角合并——正是人类逻辑推理的原子操作：**视角就是维度，而维度是逻辑演算的基础**。人类专家面对复杂问题时并不"更用力地想"，而是不断变换审视问题的维度，直到问题在某组维度下变得显然。

这背后是两条逼近答案的路线：神经网络**固定坐标系**，靠梯度下降拟合一个复合函数；逻辑推理**固定函数、变换坐标系**——不断更换维度（换基底），直到问题在合适的坐标下可解。两者殊途同归（猜想其表达能力等价，此为开放假设，见 [[棱镜推理|prism]] 第四节），但过程与原理完全不同。Unigen 试图把这两条路线接起来：神经元负责在维度内生成，逻辑负责维度的选取、变换与组合——可称之为**逻辑向 AI**。

以此观之，当前 Prism 只是维度演算的**最退化形式**：抽取维度、逐维评价、合并求精。真正的逻辑推理发生在**维度之间**——维度的组合、变换、约束传播——这正是 Loom 与 HTN 阶段要逐级落地的东西，也是这条路若走通、可能触发新范式的原因。地基的每一块砖都要敲实，所以对 Prism 的多视角实测反馈，是当前最迫切需要的。


Prism背后蕴含的其实是人类的逻辑能力（逻辑拆分再组合的过程），视角是维度，维度之间的演算就是逻辑。因此，unigen也可以称为逻辑向AI--它试图把基于Horn范式的逻辑推理与基于神经元的生成式AI衔接起来，以实现 **超长链条，超高确定性地自我进化** 的目标。因此，对于Prism的多视角评估，是最迫切需要的。
