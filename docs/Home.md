# 🎯 欢迎来到 Unigen

**Unigen（Uniform Generation）让 AI 完成复杂的、长调度的、人工质量的任务。**

从剧本到完整视频。从一个创意到一整本小说。从需求到可运行的代码。这类任务的共同点是：

- **工作流足够复杂**——几十上百个步骤，多种中间产物；
- **工作流改动频率不高**——但对质量要求极高；
- **对时间不敏感**——一次执行数天甚至数周都可以接受，**只要质量不崩**。

当前主流的 ReAct 式 Agent 不适合这个场合：执行一次太长，无法靠走一步看一步来收集反馈；而足够复杂的工作流，其质量评估本身就是一个大话题。Unigen 为此选择了一条不同的路——**规划与执行分离，代码作骨、LLM 填充，对长链条漂移零容忍。**

## 你是哪类用户？

Unigen 明确服务两类人，请从对应入口进入：

### 🎨 我是创作者——我想用工作流做出东西

你是视频创作人、小说作者、或任何想让 AI 高质量完成大活的人。你需要的是一个**灵活、可局部调整的成熟工作流**——至于它是 AI 进化出来的还是人工撰写的，你不必关心。

→ 从 [[Getting-Started|快速开始]] 和 [[Using-Workflows|运行你的第一个工作流]] 开始。

### 🧩 我是开发者——我想构建 / 进化工作流

你想为某个领域打造高确定性的长链条工作流，或者对"让规划器自动生成和改进工作流"这个方向本身感兴趣。Unigen 为你提供：

- 以**代码**为载体、可动态加载的工作流运行时；
- 内建的长链条开发方法：[[prism|Prism 多棱面批判精炼]]；
- 一条明确的规划器演进路线：[[loom|Loom 图黑板规划]] → [[planner|HTN/HDDL 符号规划]]。

→ 从 [[Developer-Guide|工作流开发总览]] 开始。

## 项目哲学：迭代靠记录，不靠猜

Unigen 的工作流是**可读、可编辑、可发布**的结构，人可以直接介入；所有执行历史通过 OpenTelemetry 兼容端点采集留存，作为人工或 AI 改进工作流的依据。

演进路线（详见 [[Roadmap|路线图]]）：

> 承载有实际价值的工作流（人工撰写为主）
> → 规划器辅助生成与改进
> → **终极目标：无人介入，全自动改进。**

## 项目状态

早期阶段。核心运行时可用：Electron 桌面应用（Windows / Linux / macOS），可从 Releases 下载免安装版。当前内建**剧本到视频**项目类型（测试与演示目的）与**空白项目类型**（面向开发者，内建工作流开发能力）。

内建推理方法可在反思助手中直接体验；
规划器按 Prism → Loom → HTN 三阶段推进中，各阶段状态见对应文档。
