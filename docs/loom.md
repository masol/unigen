
# 基于图黑板的逆向交付物规划

当前路径: docs/loom.md
状态: 概要设计完成，待实现。定位为 docs/planner.md（HTN/HDDL 路线）的
**前置过渡方案**——用极小的工程量兑现"代码作骨、LLM 填充"的核心立场，
同时为 HTN 路线沉淀图数据与实测反馈。

---

## 一、一句话定位

把"要怎么做"彻底转化为"要什么"：以**交付物（artifact）**为节点、**转化
（method）**为边，从终极交付物出发**逆向回归**，逐节点由 LLM 局部拆解、由
代码维护全局图与术语表，直至所有分支闭锁在人类原始输入上；再拓扑反转为正向
执行蓝图。**没有任何一个 LLM 见过或设计过全局——全局解在图上涌现。**

这不是新发明，是三个成熟概念的组合：
- **Backward goal regression（逆向目标回归）**：从目标反推前置，直到命中初始
  事实——经典规划的标准技法之一，对应本方案的逆向拆解主循环。
- **Blackboard Architecture（黑板架构）**：多个专家围绕共享黑板工作，各自只
  读写局部，全局解涌现。本方案的"图 + 术语表"就是黑板，各 LLM 任务（锚定/
  拆解/对齐/换路）就是专家。"没有任何 LLM 设计全体"正是黑板架构的定义性特征。
- **Prism（多棱面批判精炼，见 docs/prism.md）**：包裹唯一的重语义环节
  （单步拆解），用固定棱面拦截坏拆解。

"代码作骨、LLM 填充"因此可准确表述为：**图黑板上的 LLM 逆向回归规划**。

## 三、核心数据结构

**图（graphology 有向图）承载全部状态，同时就是术语表**：
- artifact 节点: {id(snake_case), aliases[], modality, definition(一句话语义
  锚点，对齐比对的是它而非名字), status: source|derivable|pending|blocked,
  provenance}
- method 节点: {id, transform(这步做什么,直接成为工作流文案), gate:A|B,
  status: open|closed|blocked, rejected[](被拦截的边及原因——失败不删,留作
  换路依据)}
- 边: artifact --input_of--> method --produces--> artifact
  （method 的多个 input = AND；同一 artifact 被多个 method 产出 = OR）

**图为唯一真相源**：不设独立队列。主循环每轮从图中查询 pending 节点处理并
回写（reconcile 模式）。持久化 = graph.export() 整图落盘，恢复 = 载入后照常
"找 pending"——**断点续跑免费**，所有失败尝试留在图上可审计。

## 四、两个内嵌判定关卡（在单步拆解提示词 X2 内部，非独立提示词）

- **关卡 A · 多模态桥接**：交付物为非文本（视频/音频/图像）→ LLM 不能直接
  产出字节流，必须逆推为"文本接口"依赖（视频片段 → 文生视频提示词 + 运镜
  参数），把多模态降维为文本级依赖。
- **关卡 B · 概念横向解构**：交付物为文本/人类概念 → 拆分其原子构成
  （视频提示词 → 主体+动作+运镜+风格），各子项并行递归。
- **触底**：子交付物可由人类原始输入直接推导/提取 → 该分支停止。

## 五、LLM 任务清单（共 6 个，模板固定，system+user 双提示词）

| # | 任务 | 职责 |
|---|---|---|
| X1 | 目标锚定 | 入口闸门：定终极交付物+原始素材；不可规划则拒绝 |
| X2 | 单步逆向拆解 | 核心。对一个 pending 交付物给出 1~3 个 method(OR)，内嵌关卡 A/B |
| X3 | 对齐复核 | 新词是否等同已有交付物，**比对 definition**；低置信不自动合并 |
| X4 | 阻塞换路 | 带失败原因要求换拆法；giveUp 则 blocked 向上传播（回溯引擎） |
| S2′| 固定棱面批判 | Prism 变体：完备性/可达性方向/粒度/词表贴合 四固定棱面并发审 X2 |
| S3 | 合并精炼 | 综合批判改写，changed 二值门+回退，≤2 轮 |

代码侧硬保证（不交给任何 LLM）：normalize、相似度预筛、willCreateCycle
（graphology-dag，加边前预判）、递归守门（深度上限+重复模式）、真闭包回填
（全 inputs 触底才标 derivable，禁止乐观标记）、拓扑排序与反转编译。

## 六、能承诺什么

1. **硬保证**：终止（守门）+ 合法（凡输出的蓝图必无环、依赖闭合、术语统一）。
2. **结构性保证**：OR 多候选 + blocked 传播 + X4 换路 = 带回溯的系统化搜索，
   在 LLM 知识可及的分解空间内穷尽候选，不走一步死一步。
3. **不承诺**：① LLM 不知道的路线找不到；② 客观不可达（永动机）造不出——
   但会**收敛到 blocked 并精确报告断点、已试拆法与原因**，而非幻觉一条假路。
   这份 blocked 报告正是 prism.md roadmap 阶段三（世界约束层）的种子数据。

## 七、与 HTN 路线（docs/planner.md）的关系与取舍

| | Loom（本方案） | HTN/HDDL + Prolog |
|---|---|---|
| 工程量 | 小 ：无 HDDL 编译、无符号求解器 | 大 |
| 求解 | 图上真闭包回填 + 拓扑反转（代码即可） | SLD 归结/tabling，符号级确定性 |
| 确定性 | 结构合法性有硬保证；**语义正确性依赖 X2+Prism 质量** | 语义也被前置条件/效果模型约束，确定性更高 |
| 状态/时序 | 无时间步事实链，不表达状态演化 | 时序事实链，可表达 |
| 适合 | 交付物驱动的生成流水线（剧本→视频等） | 强合规、状态敏感的长链任务 |

**取舍结论**：Loom 是 HTN 的真子集式过渡——它的图（artifact/method/OR/AND）
与 HTN 的任务网络同构，未来升级路径是把 method 附上前置/效果谓词、接入符号
求解，而已沉淀的图数据直接复用。先用 Loom 验证"代码作骨、LLM 填充"的整体
价值（含人工创建/修改工作流的价值），再决定是否为确定性支付 HTN 的成本。

## 八、Roadmap 衔接

Prism（视角层，已有初步实测）→ **Loom（结构层过渡，本文档）** → HTN/HDDL
（结构层完全体）→ 认知模型/世界约束层（开放研究）。本文档随实现与实测更新，
被证伪的判断明确标注修正。
