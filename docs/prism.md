# Prism（棱镜推理）：一种取代 CoT 的多棱面批判精炼方法

当前路径: docs/prism.md
状态: 早期实测阶段。已在若干**简单生成任务**上验证出正向效果（见第三节实测），
但复杂任务、可量化指标、跨模型规模化对比均未完成。本文既是方法说明，也是一份
公开的求证请求——欢迎用更复杂的真实任务来证伪或修正下述判断。

---

## 一、动机：CoT 的问题不在"想得少"，而在"只想了一遍"

Chain-of-Thought 让模型把推理过程写出来，确实提升了质量。但它有一个结构性弱点：
**它是单向的一条链**。链上任何一步的视角偏差、遗漏、或早期错误，都会一路带到结尾，
模型没有机会站到别的角度重看自己。它"想得更清楚"，但没有"从别的角度再想一遍"。

人类专家不是这样工作的。写完一段代码，他会分别用"语法对不对""结构合不合理""边界
情况漏没漏""性能可不可接受"几个**互不相同的视角**回头审一遍——每个视角是一次独立的、
带专门标准的检查，而不是把所有顾虑挤在一条思维链里。

Prism 把这件事显式化：**先答，再从多个正交视角分别批判，最后合并改写。**

---

## 二、方法：草稿 → 棱面拆分 → 分面批判 → 合并精炼

Prism 是一个四段式的**反思回路**（注意：不是 ReAct——它没有外部动作，全部在模型内
自省；ReAct 的质量来自外部真值，Prism 的质量来自视角的正交性与批判的独立性）：

1. **草稿（Draft）**：正常回应用户，得到初稿。等价于"先给个答案"。
2. **棱面拆分（Split）**：针对"这个问题的答案怎样才算好"，拆出 2~5 个**正交视角**
   （棱面 facet）。例如软文任务拆成 `选题角度` / `情绪钩子` / `逻辑合理性` /
   `受众贴合度`；诗歌仿写拆成 `意象` / `韵律` / `情感真挚度` / `新意`。视角由模型
   针对**当前问题**动态生成，不是固定清单。
3. **分面批判（Critique）**：每个棱面派一个"专职评审"，**只从这一个角度**审查草稿，
   给出问题与修改建议。各棱面相互独立，可并发。关键：批判同时看到**原始用户输入**
   和**草稿**，避免评审跑题或忘了用户到底要什么。
4. **合并精炼（Merge-Refine）**：汇总所有棱面的批判，改写出成稿；带一个"是否真的
   改好了"的二值门——没改好就回退初稿，不为改而改。

产物是"成稿 + 一份可读的批判/改动记录"，天然可审计。

隐喻即命名：一束白光（草稿）射进棱镜，被分解成一条光谱（多个正交视角），每条谱线
单独检视，再重新汇聚成一束更纯的光（精炼稿）。代码里用 `prism` 作代号、`facet`
指单个棱面、`draft/refined` 指草稿/成稿。

## 三、实测观察（简单任务，初步，非严格评测）

已在若干**开放式生成任务**上手工试用，包括：从产品简介生成推广软文、随口要求
仿写一段诗歌、续写小说片段等。观察如下——这些是主观印象，尚无量化指标，列出是为
求证而非下结论：

1. **低端模型被显著抬升，逼近甚至局部超越高端模型的裸输出。**
   同一个弱模型，裸答与套 Prism 后判若两人。在**某些棱面上**（尤其"选题/切入角度
   的刁钻程度""逻辑合理性"），Prism 加持的低端模型产出，甚至**超过高端模型的
   单次直答**。推测原因：这些棱面的价值来自"被强制换角度看一遍"，而不是来自模型
   的原始容量——而"换角度"正是 Prism 用流程补上的，与模型大小相对解耦。

2. **在简单任务上，高端模型与低端模型套 Prism 后差异不大——两者都很好。**
   这是最值得深究的一条。裸模型差距明显，套上 Prism 后差距被大幅抹平，双方都收敛到
   "相当好"的水平。

   一个自然的怀疑是：**是任务太简单了**。简单任务的"好答案空间"本就不大，Prism 帮
   低端模型跨过门槛后，高端模型的余量无处发挥，于是看起来"拉平"。若如此，那么在
   **足够复杂**的任务上，两者应重新分开——因为复杂任务的好答案空间更大，模型的原始
   容量会重新成为瓶颈，Prism 补的"视角"不足以替代"容量"。

   **但这只是假设。复杂任务尚未实测。** 这正是本文最想要外部反馈的地方（见第六节）。

## 四、一个更根本的怀疑：能力、复杂度、认知模型、客观世界

第三节的"拉平"现象，把我推向一个更大的问题，这里作为**开放假设**记录，我并无定论：

**猜想 A：模型能力与问题复杂度，可能是同一把尺子的两端。**
"这个模型够不够强"和"这个问题有多难"或许不是两件事，而是同一个量纲上的比较——
能力足够就解得动，不够就解不动。若成立，第三节的拉平就有了解释：简单任务落在尺子的
低端，大小模型都够得着；复杂任务落在高端，才重新拉开。

**猜想 B：问题复杂度不是问题自身的属性，而是被"认知模型"决定的。**
同一个问题，换一套认知框架，难度天差地别。"复杂度"因此不是绝对的，它相对于求解者
所持的认知模型而言。Prism 之所以能抬升弱模型，或许正是因为它**临时给模型装配了一套
更好的认知模型**（多视角、分面批判），而不是增加了模型的容量。

**猜想 C：认知模型的上界，由客观世界决定——有些输入到输出根本不成立。**
让 LLM 设计一台永动机，无论多强的模型、多好的 Prism、多刁钻的视角，都不会成功——
因为**输入与输出之间的可达性，受客观世界约束**，不因推理方法而改变。这划出了 Prism
（乃至一切纯推理方法）的硬边界：**Prism 能优化"表达"和"角度"，但改变不了"客观上
是否可达"。** 一个方法再好，也造不出客观世界不允许的路径。

把三条连起来：**客观世界 → 决定认知模型的上界 → 认知模型 → 决定问题复杂度 →
复杂度与能力同尺 → 决定可解性。** Prism 作用在"认知模型"这一层：它换来更好的视角，
所以在认知模型是瓶颈时有效；但当瓶颈落到"客观可达性"时，它无能为力。

这些是猜想，不是结论。部分我没有答案，正在研究中。列在这里，是想让 Prism 从一开始
就带着它的边界意识生长，而不是被当成万灵药。

## 五、为什么它可能比 CoT 好（推断 + 部分实测）

| 维度       | CoT                    | Prism                                        |
| ---------- | ---------------------- | -------------------------------------------- |
| 推理形状   | 单向一条链             | 一次生成 + 多条正交短链批判 + 一次汇聚       |
| 错误传播   | 早期偏差一路带到底     | 批判层作闸门，拦下一部分单链漂移             |
| 视角覆盖   | 取决于模型当下想到什么 | 显式强制覆盖多个正交角度                     |
| 弱模型抬升 | 有限                   | **实测显著**（第三节）                       |
| 可审计     | 思维链混在一起，难定位 | 每个棱面批判独立成条，改动可追溯             |
| 成本       | 1 次调用               | 1(草稿)+1(拆分)+N(棱面)+1~2(精炼)，约 N+3 次 |

核心赌注：**用"多花几次调用"换"少犯结构性错误 + 抬升弱模型下限"**。把 CoT 隐式的、
挤在一条链里的自我怀疑，改造成显式的、分角度的、可并发的批判。

## 六、适合 / 不适合，以及最想要的反馈

- **适合**：答案有明确好坏标准、且好坏由多个正交维度共同决定的开放式生成——软文、
  诗歌/小说仿写、方案设计、评审。棱面越清晰，收益越大。已初步验证。
- **不适合**：闲聊、检索事实、单一维度的简单问答。多花 N+2 次调用换不到提升，纯浪费。
  默认不套 Prism，值得时才套。
- **不能替代外部真值**：Prism 的批判来自模型自己。需要查资料/跑代码/验环境的任务，
  那是 ReAct 的活。两者正交，可叠加（ReAct 拿真值，Prism 精炼表达）。

**本文最想征集的反馈——复杂任务实测。** 目前证据都来自简单任务，第三节的"拉平"是否
在复杂任务上重新分开，直接决定第四节猜想 A/B 是否成立。欢迎在以下方向验证并回报数据：

- 长链条、强约束、多步依赖的任务（Prism 是否仍能抬升弱模型？大小模型是否重新拉开？）
- 可量化指标下的严格评测（而非本文的主观印象）。
- 反例：Prism 明显无效或反而变差的任务类型——这类反馈同样宝贵。

## 七、已知边界

1. **自评的天花板 = 模型自身能力**。模型看不出的错，多拆视角也未必看得出。Prism 降低
   的是"想得片面"，不是"根本不会"。
2. **客观不可达不可救**（第四节猜想 C）。Prism 改变不了输入到输出在客观上是否成立。
3. **视角可能不正交或凑数**。需在代码层对棱面去重、限数（≤5）。
4. **精炼可能越改越糟**。必须有"是否真的改好"二值门 + 回退，最多 1~2 轮。
5. **成本真实翻番**。方法固有代价，不是 bug；用不用取决于任务是否配得上。
6. **当前证据仅限简单任务、主观评价**。规模化、量化、复杂任务全部待补。

## 八、与既有方法的关系

Prism 不是全新发明，是一个**特定组合的通用化定位**：

- 棱面拆分 ← 借鉴 G-Eval（LLM 自拟评估维度）、Branch-Solve-Merge（分支→解→合并）
- 生成-批判-精炼循环 ← 借鉴 Self-Refine / Reflexion

新在于：把它们合成一个**像 CoT 一样即插即用的推理外壳**——任何一次生成都能被
`prism(generate)` 包一层，而不必为每个任务手写评估逻辑。G-Eval 只评不改、BSM 是任务
分解、Self-Refine 不强调视角拆分，三者都没把自己定位成"CoT 的通用替代"。Prism 补的
就是这个定位。

## 九、Roadmap：从 Prism 到受客观世界约束的认知模型

Prism 是这条路线的**起点**，不是终点。它解决"单点生成的视角贫乏"，但第四节的猜想指向
一个更大的目标：让规划本身受"认知模型"乃至"客观世界"约束。分三段推进：

- **阶段一 · Prism（当前）：视角层。**
  通用多棱面批判精炼，抬升单点生成质量，取代 CoT。作为独立方法先跑通、广泛收集实测，
  尤其复杂任务反馈。产出的批判/改动记录，天然是后续阶段的训练与参考数据。

- **阶段二 · Prism → HTN：结构层。**
  把 Prism 的"棱面拆分"从"评估视角"升格为"任务分解视角"，驱动分层任务网络（HTN）的
  method 生成与收敛（见 docs/planner.md）。此时 Prism 不再只优化一段文字，而是优化
  "一个目标该拆成哪些子目标"。**结构的硬保证（词表对齐、环检测、受控递归、AND-OR
  回溯）由代码骨架承担，Prism 只补语义**——即"留它做生成器，别让它做架构师"。

- **阶段三 · HTN → 认知模型：世界约束层。**
  为规划器引入显式的**认知模型**：输入到输出的可达性，不再只由 LLM 直觉判断，而是受
  一层客观约束校验（第四节猜想 C——永动机式的目标应在编译期被判不可达，而非反复
  生成再失败）。这层约束本质上是 HTN 前置条件/效果模型的自然延伸，最终指向"规划器
  知道什么客观上做不到"。这一阶段大部分是开放问题，尚无成熟方案。

一句话概括路线：**Prism 优化"怎么想"，HTN 约束"怎么拆"，认知模型划定"什么客观上
可达"。** 三层从软到硬，逐级把不确定性收敛给确定性。
