
## 使用类

**Q: 我是视频创作者/小说作者，需要懂规划器原理吗？**
A: 完全不需要。你只需要导入工作流、在侧边面板配置参数、运行、收获成品。
工作流是 AI 进化出来的还是人工撰写的，对你透明。

**Q: 为什么工作流是代码而不是配置文件？**
A: 为了让规划器（而不只是人）能修改它。可动态加载的代码是"人工可介入 + AI 可持续改进"两个目标的公共载体。


**Q: 一次执行要跑好几天，中断了怎么办？**
A: 支持断点续跑，执行过程会落盘，成功的模型调用，不会重复执行。


**Q: 系统要求高吗？需要 GPU 吗？**
A: 取决于模型策略。全部使用外部模型 API 时（创作者推荐），本地只有 SQLite 和 LanceDB 两个轻量存储，普通办公机即可（参考：4 核 / 8GB 内存 / 2GB 磁盘，无需 GPU）。

启用内建模型（嵌入/重排等）时，可用模型由机器配置决定，可借助[llmfit](https://github.com/AlexsJones/llmfit) 评估——注意它不直接推荐嵌入/重排模型，需自行换算。

**Q: 模型要为每个项目分别配置吗？**
A: 不用。左下角设置里配置一次（文本模型必须、嵌入模型必须、重排模型推荐），适配全部项目类型。

**Q: 现在有哪些能用的工作流？**
A: 当前内建**剧本到视频**一个项目类型，定位是测试和演示。空白项目类型面向开发者。更多实际工作流（创意→小说、小说→剧本、需求→代码等）见 [[Roadmap|路线图]]。

**Q: 支持哪些大模型？**
A: 主流模型都被支持，如果遭遇不支持的模型，请发issue.

## 贡献类

**Q: 我想贡献，从哪里开始？**
A: 两条路：① 用**复杂真实任务**实测 Prism 并回报数据（包括反例），这是当前最有价值的贡献，方向见 [[prism|棱镜推理]] 第六节；② 开发并发布有实际价值的工作流。开发环境见 [[Local-Setup|本地环境配置]]。

**Q: 协议是 AGPL-3.0，我能商用吗？**
A: 可以，但请遵守 AGPL 义务：若你基于 Unigen 提供网络服务或分发修改版，必须以同等协议开放对应源码。详情请阅读协议原文。

**Q: 开发者做好的工作流怎么给创作者用？**
A: 在反思助手中执行 `/export`，把知识中心导出为一个新的项目类型，发布后创作者导入即可使用。
