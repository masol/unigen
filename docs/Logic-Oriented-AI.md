# 🧮 逻辑向 AI：维度、坐标系与两种逼近

> **给务实主义者的建议：**  如果你不关心“坐标系”或“逻辑向”等哲学概念，只需将 Unigen 理解为一个**垂直领域的代码生成与执行平台**。我们的唯一目标是：**专门执行超长链条任务，且对状态漂移零容忍。** 一次执行可能长达数天，不同于追求通用与灵活的传统 Agent（如OpenClaw、Hermes），Unigen 的核心价值在于用确定性的逻辑骨架接管生成过程，确保长链条任务不漂移、不遗忘、不捏造。

## 一、核心直觉：复杂性可能是坐标系伪装出来的

先看数学中最古老的操作——**换元法**：

∫f(x)dx 算不动，换个变量 u=g(x)，突然变成 ∫h(u)du，一步出结果。函数没变、值没变，只是坐标系变了，问题就"消失"了。

同一个向量，在直角坐标系里是 (3, 4)，在极坐标系里是 (5, 53.13°)——**值没变，表达变了**。推广开来：一个映射 y = f(x) 在原始坐标系下可能极其复杂，但若存在变换 T，使得新坐标系下 T(y) 与 T(x) 呈简单关系，那么求解 f 就退化为寻找 T。

科学史反复上演这一幕：牛顿之前，天体运行与苹果落地是两套规则；引入"万有引力 + 微积分"这组新坐标，整个宏观世界的**定律**坍缩成 F=ma。在这些案例里，f 本身并不复杂，它只是被一个糟糕的坐标系伪装成了复杂的样子。

但这个直觉有严格的适用边界（见第四节），不能不加限定地推广。先看它在工程上引出的对比。

## 二、两种逼近答案的路线：同为"编译 + 执行"，编译的对象不同

先排除一个容易走偏的层级：有人会说"神经网络的每一层也是坐标变换，深度学习就是表示学习"。这话在**函数内部结构**的层面是对的，但与本页无关——那些逐层变换属于**被拟合出来的答案本身**，不是求解器对问题的重新表述。本页讨论的自始至终是**求解器层面**的坐标系。

在求解器层面，两条路线的结构其实是**对称的**——都分为构建（train/编译）与执行（predict/运行）两个阶段：

**神经网络：编译 = 在预先承诺的坐标系里拟合系数。** 架构定下的那一刻，参数空间的维度与度量就冻结了；训练无论多久，都只是在这个不变的高维框架内调整数字。训练期的特征自适应确实存在（这是深度学习强于固定基拟合的原因），但它被限制在初始架构的归纳偏置之内。编译一次完成，**摊销至训练分布覆盖的场景**；执行是自回归的逐 token 推演——有状态、含采样，但原生形态没有结构化的校验挂载点；产物是不可读的权重。[^2]

**逻辑：编译 = 搜索坐标变换序列本身。** 面对一个具体问题（或问题类），逻辑重新选取审视维度、重新构造概念映射——它在编译期发明本题专属的"傅里叶基"：拆出问题的分量（维度），为每个分量配上变换函数（概念映射），使重新复合后的形式极简。编译的产物就是**推理链/证明/工作流**——从复合函数视角看，这是一个度数可能很高的显式组合。执行则是把这个**已编译**序列顺序重放一遍：不含搜索，成本是线性的，产物在问题类内**可复用**。要把账算清楚：若执行中校验失败、需要回溯或改写结构，那不是执行变贵了，而是**升级回编译阶段**——搜索与回溯的潜在指数级成本，全部记在编译的账上，而编译成本正是本页标出的核心悬念（[^1]）——执行本身始终廉价。

所以真正的分水岭不在"是否做坐标变换"，也不在"哪边求解贵"——而在**编译阶段搜索什么、产物长什么样**：

|                | 神经网络（梯度下降）                                     | 逻辑推理（坐标变换编译）                                   |
| -------------- | -------------------------------------------------------- | ---------------------------------------------------------- |
| 编译的对象     | **系数**：坐标系预先承诺，训练只在其内填数字             | **坐标系本身**：搜索变换序列，直到问题在某组维度下显然可解 |
| 摊销粒度       | 一次编译，摊销至训练分布内的场景                         | 逐问题类编译，类内复用，可增量重编译                       |
| 编译产物       | 不可读的权重，只能整体重训                               | 可读的工作流/推理链，可审计、可节点级修改                  |
| 执行形态       | 自回归逐 token 推演，原生形态无结构化校验挂载点——ReAct/ToT 等外挂脚手架恰是非正式的逻辑骨架 | 顺序重放已编译工作流（节点挂校验、可分支）；校验失败则升级回编译 |
| 复杂性藏在哪   | 系数里（亿级参数的取值）                                 | 变换序列里（工作流的结构）                                 |
| 失败模式       | 局部最优、不可解释、分布外失效                           | 编译搜索不终止或爆炸；找不到好坐标系，或好坐标系被证明不存在 |

一个比喻：**梯度下降把整片地形的走法一次性压进肌肉记忆，之后闭眼走；逻辑为每座迷宫绘制一张可读的地图，之后照图走。** 前者编译一次管所有迷宫但说不出为什么这么走，后者逐迷宫制图但每一步都可指认、可纠错——而同一张图，管这一类迷宫。

注意：**两者在可计算性意义上的表达能力是平凡等价的**（神经网络与 Horn 子句均图灵完备），这不是本页的悬念——而且图灵完备是**双刃剑**：它同时意味着一般情形下的推理**不可判定**，搜索可能不终止。所以工程上刻意选用受限逻辑（Horn 子句），且只承诺"人类工程任务"这一子集（第四节的域限定就是这道围栏），不承诺覆盖任意高阶逻辑。真正的开放问题是**编译效率**，它有个更精确的名字：寻找最优的问题分解通常是 NP-hard 起步。因此赌注从来不是"找到最优坐标系"，而是"LLM 启发式能找到**够用的**坐标系"；找不到时系统必须有**降级路径**——退回更粗的维度、更保守的模板，或明确拒绝，而不是硬编译。[^1]

可以确认的是：两者各有不可替代的长处——神经元擅长维度内的生成与模式补全，逻辑擅长维度的选取、变换与一致性维护。

## 三、逻辑就是递归的坐标变换，概念就是变换函数

Prism 的"棱面"（facet）不是提示词技巧。把"这个答案好不好"拆成`逻辑合理性 / 情绪钩子 / 受众贴合度`，本质是为答案空间选了一组**坐标轴**：

- 每个视角是一个维度；
- 单一维度内的判断是简单的；
- 复杂性全部来自维度之间的关系。

所谓"换个角度看问题"，字面意思就是**坐标变换**。而逻辑的关键性质在于**可以叠加与递归**：每递归一层，问题就被提升到一个更抽象的坐标系；在合适的坐标系里，原本纠缠的变量解耦，复杂推理退化为简单推导。

再往下挖一层：**概念本身就是变换函数**。"人、狗、动物"这些名词不是标签，而是映射——每一次归类，都是把一个具体对象投影到某组概念坐标轴上。概念的内涵天然模糊，正因如此，一次恰当的类比能让人豁然开朗：那一瞬间发生的，就是坐标系突然对齐。

看三段论："凡人皆有死，亚里士多德是人，故亚里士多德有死。"推理结构简单到平凡，**真正的重负全压在"亚里士多德是人吗"这个归类判定上**——而这个判定无法形式化，它就是符号 AI 六十年迈不过去的**符号接地问题**：符号系统内部演算再精密，符号与现实的挂钩始终悬空。

Prism 的回答是分工：**把归类判定交给直觉（LLM），归类完成后，符号演算得以展开。** 但必须说明：这个分工**没有解决**接地问题，只是把它**搬运**到了 LLM 经验上最强的位置——人类高频概念的归类判定（"亚里士多德是人吗"之所以可靠，是因为语料里有海量支撑）。搬运有明确的失效区：长尾概念、训练分布外的新概念、需要实时物理环境才能判定的问题——此处 LLM 的接地依据是语料共现而非世界理解，判定失效、符号依然悬空。第四节的域限定不是修辞，正是这个失效区的显式围栏。神经元干它擅长的模糊接地，逻辑干它擅长的结构演算——这正是第五节"两条路线接起来"在最小尺度上的形态。

这个分工有一个常被视为死穴、实则是强项的性质：**逻辑保真，不产真**——若 LLM 的归类判定错了，符号演算会步步合法地把错误传导下去。但正是"步步合法"救了它：三段论逆用（modus tollens）——结论被现实证伪，而推理有效，则必是前提有错。神经网络的错误弥散在亿级参数里无从归因；逻辑链的错误会以矛盾的形式在具体节点爆出，沿链回溯即可定位到那条出错的归类判定，交还 LLM 重判——这正是科学的纠错机制：被证伪的不是逻辑，是假设。接入现实校验（ReAct 式的执行、运行、查证），逻辑链就从错误放大器反转为错误定位器。

但反转不是免费的，必须标注它的暗面：在错误被发现**之前**，逻辑会给它披上严密推理的外衣，产出比裸幻觉更有说服力的 **"理性的幻觉"**。所以承诺要说清：逻辑保证的是错误**一旦暴露即可定位**，而不是错误**不会发生**；且定位性以现实校验回路真实在场为前提——校验缺席时，逻辑骨架反而是错误的美化器。这也顺带解释"循环依赖"（用不可靠的 LLM 给逻辑编译坐标系，是否自己给自己背书）：回路在**现实处断开**——LLM 只做粒度很小的单点判定，每个判定被来自现实的信号（运行结果、查证、执行反馈）兜住，背书者是世界，不是 LLM 自身。

还有一条容易画错的边界："事实判断适用、品味判断不适用"——这条线是错的。"这段情节动人吗"同样可以被逻辑处理：拆成节奏张弛 / 悬念密度 / 人物共情点等足够细的维度后，判定同样变得可归因、可改进，像"代码不通过"一样触发回溯。品味不是逻辑的禁区。

真正的边界是：**判定标准是否构成一个函数**。"一切预测皆函数"的前提是它得是函数——同一输入给出同一输出。只要存在一致的判定者（哪怕只是某一个人的稳定偏好），"更动人"就是一个可逼近的目标：逻辑将其编译为工作流，执行、收反馈、反推前提、改进，逐步逼近该判定者的判断逻辑。反之，若标准不一致（大众褒贬不一且无聚合规则、判定者自身漂移），它在数学上根本不构成函数——不是逻辑逼近不了，而是不存在可供逼近的对象，问题本身不适定（ill-posed），逻辑对它不适用。逻辑向的边界不是"品味"，而是"非函数"。

必须说明两点：其一，维度拆解不消除判定、只是下推判定——模糊被推到更细的粒度，主张是"粒度越细，单点判定越可靠"（这正是 Prism 要实测的），但"子维度之和能预测顶层标准"本身是假设，还原可能有损；其二，"有一致标准则逻辑总能逼近"是信念不是定理——一致性保证目标存在，不保证逼近成本可接受，这又落回第二节的"编译效率"悬念。至少对人类层级的工作内容，这个信念至今未被推翻（见第四节的构造性保证）。

## 四、目标定位：无幻觉的超长链条，以及为什么它可达

先把目标钉死。当前 AI 的真实短板不是"不够聪明"，而是无法无幻觉地执行超长链条任务：写一部小说，先不论好坏，是个人就能写完且前后自洽，AI 写出来的却达不到人类基础水平——人物设定漂移、情节前后冲突、凭空捏造。算术原因很简单：自回归生成的单步可靠性设为 p，n 步链条的无错概率是 p^n，链条一长，幻觉必然爆发。逻辑骨架的作用，是把"一错到底的连乘"改造成"每个节点可检、可修、可回溯的循环"——这正是第二节表格里"执行形态"一行的分水岭：原生自回归执行没有结构化的校验挂载点，而工作流的每个节点都是挂载点。无幻觉/长链条/一致性维护恰是逻辑的主场。**逻辑向 AI 的目标就是：由逻辑 harness LLM，达成无幻觉的超长链条执行。**

注意目标的措辞：是"无幻觉"，不是"写得好"。这个承诺必须拆成三级，各自的条件不同：

1. **结构一致性**（设定不漂移、前后不冲突）——**无条件承诺**：一致性可以内部自检，不需要外部判定者；
2. **事实正确性**（不凭空捏造）——**条件承诺**：条件是存在可查证的 oracle（可运行的代码、可核对的资料、可执行的反馈）。必须承认：**自洽的捏造**——内部完全一致、但与事实不符的虚构——能通过一致性自检，只有外部校验能拦住它。这是 LLM 概率本质决定的，逻辑骨架不消除它，只保证它一旦暴露可被定位（第三节）；
3. **错误可定位性**——**无条件承诺**：无论哪类错误，只要暴露，就能沿工作流回溯到具体节点的具体判定——这是相对神经网络的净增益，也是"无幻觉"一词在本页最严格的含义：不是承诺永不出错，而是承诺**错误不静默**。

"写得好"则是另一档的条件承诺——它需要一个一致的外部判定标准，有，则逻辑可将"更好"编译为工作流逐步逼近（第三节）；没有（标准本身不构成函数），则任何方法都无法逼近——不只是逻辑。目标的边界与方法的能力边界精确重合，这是这条路线自洽的证据，不是巧合。

然后划清本方法不承诺什么。有一个更大的问题——"能否压缩宇宙的一切复杂性"——答案已被定理钉死为悲观，但那不是本页的问题，只需列作边界：

- **任意函数不可尽简**：计数论证（Shannon, 1949）表明几乎所有函数不存在任何简化表示，"总能找到坐标系"作为全称命题为假；
- **简化定律 ≠ 简化求解**：牛顿简化了定律，三体问题仍无闭式通解（Poincaré/Bruns 定理级否定）；混沌系统规则极简而长期行为计算不可约。坐标变换能简化"世界的规则"，不保证能简化"规则的后果"；
- **被期望的坐标系可以被证明不存在**：爱因斯坦相信量子怪异只是坐标系没找对（局域隐变量），贝尔定理加实验证明该坐标系不存在。

这些定理圈掉的问题，本就不在目标之内——我们不指望 AI 求解人类智能解不了的问题；人能干的，才让它干。而一旦把问题域限定为"人类问题"，形势就完全反转：

**人类问题域自带存在性保证。** 写文案、做规划、写小说、生成代码，全是人类用概念构造出来、且已经反复解决过的问题——人类每一次成功求解，都是"好坐标系存在"的一次构造性证明。在这个域上，"存在让问题变简单的坐标系"不是赌博，是已兑现的事实。

但要拆开两件事：**存在 ≠ 能廉价找到**。人类找到过坐标系，只证明解存在；而最优分解是 NP-hard 起步（第二节），所以本路线赌的不是"LLM 总能找到最优坐标系"，而是"LLM 是目前最具性价比的启发式，且在多数人类工程任务上能找到**够用的**坐标系"——这仍是开放假设，正是 Prism 实测（第六节）存在的全部理由；假设失败时，走第二节声明的降级路径，而非在坏坐标系上硬编译出貌似严密的垃圾。

## 五、Unigen 的位置：把两条路线接起来

Unigen 的"代码作骨、LLM 填充"，在本页的语言里可以重述为：

> **神经元负责维度内的计算，逻辑负责维度间的计算。**

即：让擅长模式补全的神经网络在单一坐标轴内做判断与生成，让擅长结构操作的逻辑系统负责坐标轴的选取、变换与组合。这就是"逻辑向 AI"的含义。

按第二节的"编译/执行"两阶段看，**工作流就是逻辑侧的编译产物**——它是坐标变换序列的工程形态。围绕它有两组各两条的路线：

**如何得到工作流（编译）**：

1. 直接让 LLM 设计工作流——一次前向吐出结构，便宜，但结构本身可能带幻觉；
2. 借助逻辑构建工作流——维度拆解、约束传播、符号演算，昂贵，但结构可审计。

**如何运行工作流（执行）**：

1. ReAct 式 LLM 自循环——步骤边界模糊，校验依赖模型自觉，链条一长回到 p^n；
2. 逻辑驱动/保障的多步骤执行——步骤边界清晰，每个节点挂校验，天然支持长链、对抗幻觉。

Unigen 的选择是**编译、执行两端都靠逻辑侧**，这不是排斥 LLM，而是把 LLM 放回它擅长的位置：编译期充当坐标系搜索的启发式（第六节），执行期充当节点内的生成与接地判定（第三节）。对照路线图逐级看：

| 阶段             | 维度演算的形态                                                               |
| ---------------- | ---------------------------------------------------------------------------- |
| Prism（现在）    | **最退化形式**：抽取维度 → 逐维独立评价 → 合并。维度之间只有"汇总"这一种运算 |
| Loom             | 维度获得**结构**：交付物图上的 AND/OR、依赖、回溯——维度间开始有约束传播      |
| HTN/Prolog       | 维度间演算交给**符号引擎**：合一、归结、回溯，Horn 子句是维度关系的形式载体  |
| 认知模型（远期） | 维度演算受**客观世界约束**：哪些坐标变换在现实中根本不存在                   |

## 六、为什么说 Prism 是基石

如果维度间演算是大厦，Prism 就是承重测试：它验证的是最基本的一件事——**LLM 能否针对具体问题产出近似正交、贴题的维度，并在单一维度内可靠判断。**

用本页的语言说：Prism 测试的是 LLM 能否充当"坐标系编译"的廉价启发式——这正是第二节标出的真正开放问题。这两个能力若不成立，Loom 的图和 HTN 的子句都是空中楼阁；若成立（初步实测正向，见 [[棱镜推理|prism]] 第三节），后面的每一层都是在给"维度"这个原语增加更复杂的运算。

这就是我反复纠缠这个"小技巧"、并把多视角实测列为最迫切需求的原因。

## 七、边界

本页哪些是确认的、哪些是信念、哪些已被否定：

### 定理或实测支撑

- 视角拆分能显著抬升弱模型的生成质量（简单任务实测，[[prism.md|prism]] 第三节）；
- 坐标变换求解在数学上有大量成熟对应物：换元法、重参数化、自然梯度、等变网络——"换对坐标系可以绕过暴力拟合"在这些局部场景是被证明的；
- 长链条自回归生成的错误按 p^n 复利累积——幻觉在长任务上爆发有算术必然性；
- 逻辑链上的错误**一旦暴露**可经 modus tollens 回溯定位到具体前提（定位性是无条件的，暴露依赖现实校验在场，第三节）；
- 人类问题域上"好坐标系存在"有构造性保证（人类已反复解决过这些问题，第四节）；
- 神经网络与 Horn 子句在可计算性意义上均图灵完备，"表达能力"不是两条路线的分水岭；
- HTN 分解与 Prolog SLD 归结的结构同构（经典规划文献，[[planner.md|planner]] 第六节）。

### 开放假设（可能被证伪，欢迎攻击）

- 好坐标系不仅存在，且 LLM 能充当**目前最具性价比的**编译启发式，在多数人类工程任务上找到够用（而非最优）的坐标系——最优分解是 NP-hard 起步，此为第二节的核心悬念、Prism 实测对象；
- LLM 的归类判定（符号接地）在**人类高频概念**上足够可靠，且长尾/分布外的失效能被域限定围栏与现实校验回路兜住（第三节）；
- LLM 生成的工作流是否天然具有足够高的局部可修复性（repairability），使得"逻辑保真、现实校验、局部回溯"能够形成一个稳定收敛的闭环(对抗逻辑不产真)。
- 现实校验回路的覆盖密度足以让"理性的幻觉"（自洽的捏造）在传播成灾前暴露（第三、四节）；
- 逻辑编译出的工作流在其问题类内的复用收益，足以摊销高昂的编译成本（含回溯升级的成本，工作流的泛化边界待实测）；
- 问题复杂度相对于认知模型而非问题自身（[[prism.md|prism]] 猜想 B）；
- 这条路线能走到"超长链条、超高确定性的自我进化"。

### 不在目标内 / 已被定理圈掉（本方法不承诺）

- 人类智能解不了的问题（第四节三条定理级否定：任意函数不可尽简、简化定律≠简化求解、特定坐标系可证不存在）；
- 判定标准不构成函数的问题——同一输入无一致输出（大众褒贬不一且无聚合规则、判定者自身漂移），此时不存在可供逼近的对象，方法不适用。注意边界画在"非函数"而不是"品味"：有一致判定者的品味目标在射程之内（第三节）；
- 客观世界不允许的输入→输出映射（[[prism.md|prism]] 猜想 C）。任何维度变换都造不出永动机——这是硬边界。

[^1]: 更精确地说：可计算性等价是平凡的，但它对成本沉默——且图灵完备的另一面是不可判定，这就是工程上选用受限逻辑（Horn 子句）并限定问题域的原因。逻辑向 AI 成立的前提不是"逻辑能表达神经网络能表达的一切"（这已知为真且无用），而是"坐标系编译存在比暴力拟合更廉价的启发式"——鉴于最优分解是 NP-hard 起步，这里的"找到"指找到够用的、而非最优的坐标系；若启发式失败，系统降级而非硬编译。Prism 实测的正是这个前提。

[^2]: 链式思考（CoT）让 LLM 看起来也能"现场换框"，但那个换框动作本身是冻结权重内的模式补全——它在**模仿**逐题换框，而非在架构层面**执行**它：没有显式的坐标系表示，没有一致性维护，没有可回溯的变换序列。ReAct/ToT 等外挂框架为它补上了部分校验与分支能力，但这些脚手架本身就是**非正式的逻辑骨架**——业界自发走向 harness，恰是"原生执行缺校验挂载点"的旁证。Unigen 做的是把这层骨架形式化、显式化、结构化。
