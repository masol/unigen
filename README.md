# Unigen — Uniform Generation

**Unigen 是一个专为“超长链条、人工级质量”任务设计的零漂移执行平台。**

当你的任务是从剧本到成片、从创意到整本小说、从需求到可运行代码时，你需要对抗的不是算力，而是**时间带来的熵增（漂移）**。

Unigen 的核心承诺是：**对漂移零容忍**。不同于追求通用与灵活的 Agent（如 OpenClaw），Unigen 用确定性的逻辑骨架接管生成过程，确保在数天甚至数周的长链条执行中，状态不丢失、事实不捏造、逻辑不崩塌。

## Unigen 适合谁

| 你是 | Unigen 给你什么 |
| :--- | :--- |
| **创作者 / 使用者**（视频创作人、小说作者……） | **交付成品，而非工具。** 开箱即用的高质量工作流：新建项目、点击运行、收获成品。你不需要关心工作流是人写的还是 AI 进化出来的，你只需要结果。 |
| **工作流开发者** | **构建“绝不漂移”的长链条能力。** 一个以代码为载体、可动态加载的运行时，加上一套内建的长链条开发方法，帮你构建步骤极多但质量极高的工作流，并发布给创作者。 |

## Unigen 不适合什么

- **对话式、即时响应的 Agent 任务**——那是 ReAct 类框架（如 OpenClaw）的主场，Unigen 刻意不做。
- **简单的 3~5 步自动化**——Unigen 的确定性机制对短任务是纯粹的负担。
- **需要边执行边探索的开放任务**——Unigen 假设工作流复杂但改动频率低、质量要求极高。工作流的高动态改动，理论可行，但以当前算力程度，实践中无法容忍。

## 安装

Electron 桌面应用[^unigen-tauri]，支持 **Windows / Linux / macOS**：

- **推荐**：从 [Releases](../../releases) 下载对应平台的**免安装版**，解压即用。
- **从源码构建**：见 wiki 的 [本地环境配置](../../wiki/Local-Setup)（提供 Ubuntu 完整步骤）。

**系统要求**：

- 若全部使用外部模型 API（禁用内建模型），本地只有 SQLite 与 LanceDB 两个轻量存储，普通办公机即可流畅运行（参考下限：4 核 CPU / 8GB 内存 / 2GB 磁盘，无需 GPU）——**创作者推荐此方式**。
- 若启用内建模型（嵌入/重排等），可用模型取决于你的机器配置，详见 wiki [快速开始](../../wiki/Getting-Started)。

首次启动只需配置一次模型（左下角设置），即适配全部项目类型。

## 核心理念

1.  **零漂移执行**——长链条任务的头号杀手是“状态漂移”。Unigen 用逻辑骨架强制约束 LLM，确保执行数天后依然如初。
2.  **工作流即代码（蓝图）**——结构由代码硬保证，语义由 LLM 填充。规划器通过修改蓝图来改进工作流，而非靠“猜”。
3.  **规划与执行分离**——执行器负责跑，规划器负责编译合法路径。
4.  **迭代靠记录**——执行历史全量可追溯（OpenTelemetry 兼容），每一次改进都基于真实运行数据。
