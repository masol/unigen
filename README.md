# Unigen — Uniform Generation

**Unigen 是一个面向"复杂长链条、人工级质量"任务的工作流平台：当你需要 AI 完成
从剧本到成片、从创意到整本小说这类步骤极多、质量要求极高、但不赶时间的工作时
（一次执行数天甚至数周都可接受，只要质量不崩），Unigen 负责稳定地承载这样的
工作流，并用规划器让工作流本身持续进化——今天靠人工与 AI 协作
改进，终点是无人介入的全自动改进。**

## Unigen 适合谁

| 你是                                                      | Unigen 给你什么                                                                                                                                                                  |
| --------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **创作者 / 使用者**（视频创作人、小说作者、量化研究者……） | 开箱即用的高质量工作流：新建项目、点击运行、收获成品。你不需要关心工作流是人写的还是 AI 进化出来的。                                                                             |
| **工作流开发者**                                          | 一个以代码为载体、可动态加载的工作流运行时（蓝图），加上一套内建的长链条高确定性开发方法，帮你构建"步骤很多但绝不漂移"的工作流，并发布给创作者。 |

## Unigen 不适合什么

- **对话式、即时响应的 Agent 任务**——那是 ReAct 类框架的主场，Unigen 刻意不做。
- **简单的 3~5 步自动化**——Unigen 的确定性机制对短任务是纯粹的负担。
- **需要边执行边探索的开放任务**——Unigen 假设工作流复杂但改动频率低、质量要求极高。工作流的高动态改动，理论可行，但以当前算力程度，实践中无法容忍。

## 安装

Electron 桌面应用[^unigen-tauri]，支持 **Windows / Linux / macOS**：

- **推荐**：从 [Releases](../releases) 下载对应平台的**免安装版**，解压即用。
- **从源码构建**：见 wiki 的 [本地环境配置](../wiki/Local-Setup)（提供 Ubuntu 完整步骤）。

**系统要求**：若全部使用外部模型 API（禁用内建模型），本地只有 SQLite 与
LanceDB 两个轻量存储，普通办公机即可流畅运行（参考下限：4 核 CPU / 8GB 内存 /
2GB 磁盘，无需 GPU）——**创作者推荐此方式**。若启用内建模型（嵌入/重排等），
可用模型取决于你的机器配置，详见 wiki [快速开始](../wiki/Getting-Started)。

首次启动只需配置一次模型（左下角设置），即适配全部项目类型。


## 核心理念（一分钟版）

1. **工作流即代码（蓝图），可动态加载**——所以人和规划器都能修改它：
   规划器正是通过修改蓝图来改进工作流的。
2. **规划与执行分离**——执行器负责跑，规划器负责编译合法路径。
3. **代码作骨、LLM 填充**——结构（依赖、词表、无环）由代码硬保证，语义由
   LLM 局部填充。长链条漂移零容忍。
4. **迭代靠记录，不靠猜**——执行历史可追溯（OpenTelemetry 兼容采集），
   每一次改进都基于真实运行数据。
